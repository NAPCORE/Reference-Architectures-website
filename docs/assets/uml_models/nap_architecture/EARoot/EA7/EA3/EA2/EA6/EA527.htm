<b>6.2 “Harvesting functionalities”</b> assesses the ability of a NAP (National Access Point) to facilitate interoperability through the integration of metadata and/or datasets with other data portals and platforms. This involves the use of harvesting protocols and interfaces that enable the automated export of its metadata or datasets to external systems, and/or the import of metadata or datasets from external systems for integration and indexing. A value equal to 0 points out the absence of any harvesting functionalities. On the other hand, a value equal to 1 indicates the support of one-way harvesting functionalities, while a value equal to 2 indicates the support of two-way harvesting functionalities.<br/><br/>Values:<br/><b>0:</b> Absence of harvesting functionalities<br/><b>0: </b>Support of harvesting functionalities through dedicated protocols or Application Programming Interfaces<br/><b>Value+1</b> (for each of the following options): Which kind of support is provided?<br/><b>a.</b> Provision of harvesting interface or protocols for exporting functionalities (e.g., RESTful API, CSW, CKAN), allowing other portals to query and index metadata<br/><b>b.</b> Existence of harvesting interface for integration (e.g., indexing) of data sets from other portals<br/><br/>Minimum acceptable value for interoperability: 1<br/>
