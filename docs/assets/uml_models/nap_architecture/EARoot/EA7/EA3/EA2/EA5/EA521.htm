<b>Data visualization: </b>This KPI assess the extent to which a NAP can support the visualization of its exchanged content. This means extended view on dataset information. Visualization options are divided according to the form of visualization, namely numbers and graphs, map outputs, or animations (that are typically relevant for time series of data).<br/><br/><i>Note 1: Data visualization means that NAP is providing visual information about data, e.g., showing the traffic flow, speed or congestions along the time or along the road network, map visualization with location of truck parking locations etc.</i><br/><i>Note 2: Even if no data is provided via NAP interface (like in metadata directories), it is still possible to have visualization available, e.g., through WMS.</i><br/><br/>Values:<br/><b>0: </b>No visualization of data<br/><b>0:</b> Visualization is available<br/><b>Value+1</b> (for each of the following options): What types of data visualization functionalities are available? You can select multiple options:<br/><b>a. </b>Visualization of numbers and charts derived from data records<br/><b>b.</b> Visualization of data on a map (may include location points, lines or areas)<br/><b>c. </b>Visualization can use animation of “a” or “b” options above, e.g., an animation or video for a time series of a selected period<br/><br/>Minimum acceptable value for interoperability: 1<br/>
