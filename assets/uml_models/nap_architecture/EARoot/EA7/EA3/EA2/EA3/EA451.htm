<b>“Monitoring and evaluation”</b> KPI assesses whether a NAP has implemented technical procedures for monitoring and evaluating its content. The value of this KPI is increased by 1, when a NAP complies with one of the following parameters. The first parameter involves the monitoring of the visibility of a NAP with counting the access, counting the number of search requests and/or the number of registered users. The second one involves the collection of statistics regarding the consumption of the datasets at a data platform NAP. The third one involves the measurement of the performance of the system (e.g., downtime, consequences for other systems, etc.), while the final one involves the assessment of the usefulness of a NAP (e.g., qualitative feedback on the re-use of the provided data, rating of quality, surveys, etc.). It is assumed that the current KPI is not related to the minimum acceptable level of service of a NAP.<br/><br/><b>0: </b>No<br/><b>0:</b> Yes<br/><b>Value+1</b> (for each of the following options): Which technical procedures are established to monitor and evaluate the success and impact of the NAP? You can select multiple options:<br/><b>a. </b>Counting the access to the NAP, the number of search requests and/or the number of registered users<br/><b>b.</b> Collecting statistics on the consumption of datasets (e.g., downloads, subscriptions, page views)<br/><b>c. </b>Measuring performance of the system (e.g., downtime, consequences for other systems, etc.)<br/><b>d. </b>Measuring usefulness of the NAP (e.g., qualitative feedback on re-use, rating of quality, surveys, etc.)<br/><br/>Minimum acceptable value for interoperability: 1<br/><br/>
