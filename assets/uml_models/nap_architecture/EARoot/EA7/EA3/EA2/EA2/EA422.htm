<b>“Security – providers verification”</b> KPI assesses the extent to which the authenticity of provided data (e.g., input provided during the registration process) is verified. In case that a self-verification process is in place for data providers, this KPI takes a value equal to 1. On the other hand, when the authenticity of provided, data is doubled checked by both data providers and NAP operator, this KPI gets a value equal to 2.<br/><br/>Note: This is about metadata of a data provider regarding registration process (i.e., before obtaining username and password or other identity). It’s about trust/credibility of a provider<br/><br/>Values:<br/><b>N/A: </b>No data provider can use the NAP interface<br/><b>0: </b>No<br/><b>1: </b>Self-authentication of data providers (e.g., via e-mail)<br/><b>Value+1: </b>for each of the following options:<br/><b>a.</b> Authentication of data providers is double checked, i.e., by the data provider and by the NAP operator (e.g., via checking a state registry of legal bodies)<br/><b>b. </b>Usage of two-factor authentication and/or ID certificate (e.g., self-logging with secure national ID)<br/><br/>Minimum acceptable value for interoperability: 2<br/><br/>
